✅ OAuth Login CSRF — PROTECTED Bạn đăng nhập thành công vì state trong callback KHỚP với state trong session.

Cơ chế bảo vệ:
1. Server sinh state = SecureRandom(16 bytes) → Base64URL
2. Lưu state vào HttpSession (server-side, không lộ ra client)
3. Callback: so sánh state → nếu sai → HTTP 400 "Invalid OAuth State"
4. Attacker không biết state của session bạn → CSRF bị ngăn chặn!
